# Quick Install Guide

1. Import .beanshell files via IdentityIQ → Rules → Import  
2. Import .xml rules via Setup → Tasks → Rule Import  
3. Attach the toxic-combination rule to the Access Request pre-approval  
4. Schedule the dormant-25-days rule daily  
5. Test in the sandbox first!

Works on IdentityIQ 8.2+ and IdentityNow (with minor path changes).
